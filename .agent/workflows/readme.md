# Workflow: Update README.md

Ты — технический писатель, который следит за актуальностью главной документации проекта. При вызове `#readme` ты обязан актуализировать `README.md` на основе текущего состояния кода.

## Триггер
Команда `#readme` в сообщении пользователя.

## Шаги

### 1. Собрать актуальные метрики
Запусти команды для получения свежих данных:
```bash
make test-coverage      # Для метрик покрытия и количества тестов
find . -type f | wc -l  # Для общего количества файлов (исключая vendor и .git)
cat Makefile | grep '^[a-zA-Z0-9_-]*:' # Для списка команд Makefile
```

### 2. Анализ структуры проекта
Проверь ключевые папки на наличие новых компонентов (UseCases, Repositories, Rules и т.д.):
- `application/`
- `domain/`
- `infrastructure/`
- `presentation/`

### 3. Обновление документа документа
Обнови следующие секции в `README.md`:

- **Метрики (Metrics)**: Обнови процент покрытия, количество тестов, количество файлов.
- **Команды (Commands)**: Если в `Makefile` появились новые команды, добавь их описание.
- **Структура (Structure)**: Если архитектура изменилась (например, добавились правила статического анализа), отрази это в дереве проекта.
- **Технологии (Tech Stack)**: Обнови версии или добавь новые инструменты (например, PHPStan Custom Rules).

### 4. Проверка стиля
- Используй профессиональный технический русский язык.
- Соблюдай иерархию заголовков.
- Код-блоки должны иметь правильную подсветку синтаксиса.

### 5. Финализация
После обновления запусти линтер для документации, если он есть, или просто убедись, что ссылки работают.
