# Skill: README (метрики)

Обновление только цифровых данных в README.md. Текстовые описания, архитектурные блоки и примеры кода НЕ трогать.

## Шаги

### 1. Собрать метрики

```bash
make test-full  # количество тестов, assertions, покрытие, runtime
make test-e2e   # количество E2E-тестов
```

`make infection` **не запускать**. MSI/Mutation обновлять **только по данным пользователя**.

### 2. Обновить секции

- 🧪 Тестирование и покрытие кода - таблица с цифрами (Tests, Assertions, Coverage, Runtime)
- 📊 Статистика проекта - бейджи (Source Code, Test Code, Files)
- 🛠 Технический стек - версии PHP, БД, инструменты (если изменились)

### 3. Валидация

Запусти `bin/validate-docs` для проверки метрик и ссылок.

### 4. Что НЕ трогать

- Текстовые описания и формулировки
- Архитектурные блоки и примеры кода
- Дерево файлов (Project Structure)
- Другие файлы документации

### 5. Финализация

⛔️ **НЕ ДЕЛАЙ КОММИТ**. Оставь файлы изменёнными. Пользователь сам решит когда коммитить.
